## How to judge each atom

Return a result for EVERY atom, by its sub_type:

- **ui_label**: TRUE if the screen inventory clearly shows the labelled element/value; FALSE if the inventory clearly shows it absent or different; UNCERTAIN if the inventory doesn't cover that region.
- **functional**: judge ONLY against observed behavior. Do NOT mark FALSE just because the descriptive wording isn't shown on screen — a functional description is not on-screen label text. UNCERTAIN if the behavior wasn't exercised in the capture.
- **visual**: requires a screenshot. If a screenshot is provided above, judge from it. If NO screenshot is provided, you MUST return UNCERTAIN — never infer a visual attribute (color, solid vs dashed line, exact position) from text alone.
- **nav**: judge against the navigation trace provided above. TRUE if the documented path's controls existed and led to the stated states in order; FALSE if a documented control was absent or a step led to a different intermediate state than the claim asserts; UNCERTAIN if no trace is available.

Bias for every atom: a missed real contradiction is the costliest error, so only mark TRUE when the evidence actually supports the atom — but never mark FALSE merely because evidence is absent; absence of evidence is UNCERTAIN.
